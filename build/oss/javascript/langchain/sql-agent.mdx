In this tutorial, you will walk through how to build an agent that can answer questions about a SQL database. You'll build this in two ways. The first will use `create_agent` to build an agent with minimal code. The second will build a customized workflow using LangGraph. 

At a high level, the agents will:

1. Fetch the available tables and schemas from the database
2. Decide which tables are relevant to the question
3. Generate a query based on the question and information from the schemas
4. Safety-check the query to limit the impact of LLM-generated queries
5. Execute the query and return the results
6. Correct mistakes surfaced by the database engine until the query is successful
7. Formulate a response based on the results

<Warning>
"Security note"
    Building Q&A systems of SQL databases requires executing model-generated SQL queries. There are inherent risks in doing this. Make sure that your database connection permissions are always scoped as narrowly as possible for your agent's needs. This will mitigate, though not eliminate, the risks of building a model-driven system.
</Warning>

### Configure the database

You will be creating a SQLite database for this tutorial. SQLite is a lightweight database that is easy to set up and use. We will be loading the `chinook` database, which is a sample database that represents a digital media store.
Find more information about the database [here](https://www.sqlitetutorial.net/sqlite-sample-database/).

For convenience, we have hosted the database (`Chinook.db`) on a public GCS bucket.

```js
import { existsSync } from "node:fs";
import fs from "node:fs/promises";
import path from "node:path";

const url = "https://storage.googleapis.com/benchmarks-artifacts/chinook/Chinook.db";
const localPath = path.resolve("Chinook.db");

if (existsSync(localPath)) {
  console.log(`${localPath} already exists, skipping download.`);
} else {
  console.log(`Downloading ${url} ...`);
  const resp = await fetch(url);
  if (!resp.ok) throw new Error(`Failed to download DB. Status code: ${resp.status}`);
  const buf = Buffer.from(await resp.arrayBuffer());
  await fs.writeFile(localPath, buf);
  console.log(`File downloaded and saved as ${localPath}`);
}
```

### Tools for database interactions

We'll use `run` from ` SQLDatabase` to execute commands with an `execute_sql` tool. Before running the command, we'll do a check to check the LLM generated command in ` _safe_sql`

<Accordion title="Show _safe_sql details">


```js

const DENY_RE = /\b(INSERT|UPDATE|DELETE|ALTER|DROP|CREATE|REPLACE|TRUNCATE)\b/i;
const HAS_LIMIT_TAIL_RE = /\blimit\b\s+\d+(\s*,\s*\d+)?\s*;?\s*$/i;

function _safe_sql(q) {
  let query = String(q ?? "").trim();

  // block multiple statements (allow one optional trailing ;)
  const semis = [...query].filter((c) => c === ";").length;
  if (semis > 1 || (query.endsWith(";") && query.slice(0, -1).includes(";"))) {
    return "Error: multiple statements are not allowed.";
  }
  query = query.replace(/;+\s*$/g, "").trim();

  // read-only gate
  if (!query.toLowerCase().startsWith("select")) {
    return "Error: only SELECT statements are allowed.";
  }
  if (DENY_RE.test(query)) {
    return "Error: DML/DDL detected. Only read-only queries are permitted.";
  }

  // append LIMIT only if not already present
  if (!HAS_LIMIT_TAIL_RE.test(query)) {
    query += " LIMIT 5";
  }
  return query;
}

```


</Accordion>


```js
import { tool } from "langchain"
import { z } from "zod";

const execute_sql = tool(
  async ({ query }) => {
    const q = _safe_sql(query);
    if (q.startsWith("Error:")) return q;
    try {
      const result = await db.run(q);
      return typeof result === "string" ? result : JSON.stringify(result, null, 2);
    } catch (e) {
      return `Error: ${e?.message ?? String(e)}`;
    }
  },
  {
    name: "execute_sql",
    description: "Execute a READ-ONLY SQLite SELECT query and return results.",
    schema: z.object({
      query: z.string().describe("SQLite SELECT query to execute (read-only)."),
    }),
  }
);

```


## 3. Using create_agent

`create_agent` will create a [ReAct agent](https://arxiv.org/pdf/2210.03629). This is a perfect match for our task. The agent will interpret the request and generate a SQL command. The tools will check the command for safety and then try to execute the command. If the command has an error, the error message is returned to the model. The model can then examine the original request and the new error message and generate a new command. This can continue until the LLM generates the command successfully or reaches an end count. This pattern of providing a model with feedback - error messages in this case - is very powerful. 

Let's initialize the agent. To customize the agent's behavior, you write a descriptive system prompt.


```js
import { SystemMessage } from "@langchain/core/messages";
import { createAgent } from "langchain";

const SYSTEM = `You are a careful SQLite analyst.

Authoritative schema (do not invent columns/tables):
${SCHEMA}

Rules:
- Think step-by-step.
- When you need data, call the tool \`execute_sql\` with ONE SELECT query.
- Read-only only; no INSERT/UPDATE/DELETE/ALTER/DROP/CREATE/REPLACE/TRUNCATE.
- Limit to 5 rows unless user explicitly asks otherwise.
- If the tool returns 'Error:', revise the SQL and try again.
- Limit the number of attempts to 5. 
- If you are not successful after 5 attempts, return a note to the user.
- Prefer explicit column lists; avoid SELECT *.
`;

```


Now, let's create our agent with the model, tools and prompt we have set up.




```js
const agent = await createAgent({
  model: "openai:gpt-5",
  tools: [execute_sql],
  prompt: new SystemMessage(SYSTEM),
});

```


## 4 Run
Let's run this agent on a sample query and observe its behavior:



```js
function printLast(state) {
  const m = state.messages?.at(-1);
  if (!m) return;
  const role = m._getType?.() ?? m.getType?.() ?? "msg";
  console.log(`${role}: ${typeof m.content === "string" ? m.content : JSON.stringify(m.content)}`);
}
const question = "Which genre, on average, has the longest tracks?";
for await (const step of await agent.stream(
  { messages: [{ role: "user", content: question }] },
  { streamMode: "values" }
)) {
  printLast("values" in step ? step.values : step);
}
```
