## Overview

Messages are the fundamental unit of communication in LangChain. They represent
the input and output of conversations, carrying both content and metadata that
enables stateful multimodal interactions.

Messages are structured objects that contain:

* <Icon icon="user" size={16} /> [**Role**](#message-types) - Identifies the
message type (e.g. `system`, `user`)
* <Icon icon="folder-closed" size={16} /> [**Content**](#content) - Types
of content: text, images, audio, documents, and more
* <Icon icon="tag" size={16} /> [**Metadata**]() - Optional fields such as response
information, message IDs, and token usage

LangChain provides a unified message format that works across all model
providers, handling differences automatically.

## Basic usage

<CodeGroup>
```python Create and use messages
from langchain.chat_models import init_chat_model
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage

model = init_chat_model("openai:gpt-5-nano")

system_msg = SystemMessage("You are a helpful assistant.")
human_msg = HumanMessage("Hello, how are you?")

# Use with chat models
messages = [system_msg, human_msg]
response = model.invoke(messages)  # Returns AIMessage
```
</CodeGroup>

### Text prompts vs message prompts

LangChain supports two main approaches for interacting with models:

<Tabs>
    <Tab title="Text Prompts" icon="text">
        Text prompts are strings - ideal for straightforward generation tasks
        where you don't need conversation history.
        
        ```python
        response = model.invoke("Write a haiku about spring")
        ```
        
        **Use text prompts when:**
        * You have a single, standalone request
        * You don't need conversation history
        * You want minimal code complexity
    </Tab>
    <Tab title="Message Prompts" icon="messages">
        Message prompts are arrays of structured messages with roles.
        
        ```python
        from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

        messages = [
            SystemMessage("You are a poetry expert"),
            HumanMessage("Write a haiku about spring"),
            AIMessage("Cherry blossoms bloom...")
        ]
        response = model.invoke(messages)
        ```
        
        **Use message prompts when:**
        * Managing multi-turn conversations
        * Working with multimodal content (images, audio, files)
        * Including system instructions
    </Tab>
</Tabs>

### OpenAI format compatibility

LangChain messages work seamlessly with OpenAI's [dictionary format]():

<CodeGroup>
```python OpenAI dictionary format
openai_format = [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"},
]

response = model.invoke(openai_format)
```

```python LangChain message objects
langchain_format = [
    SystemMessage("You are a helpful assistant."),
    HumanMessage("Hello!"),
]

response = model.invoke(langchain_format)
```
</CodeGroup>

Both formats produce identical results. LangChain automatically handles the
conversion between formats when communicating with different providers.

## System message

A `SystemMessage` primes the model's behavior and provide conversational
context. They set the tone, define the model's role, and establish guidelines
for responses.

<CodeGroup>
```python Basic instruction
system_msg = SystemMessage("You are a helpful coding assistant.")

messages = [
    system_msg,
    HumanMessage("How do I create a REST API?")
]
response = model.invoke(messages)
```

```python Detailed persona
system_msg = SystemMessage("""
You are a senior Python developer with expertise in web frameworks.
Always provide code examples and explain your reasoning.
Be concise but thorough in your explanations.
""")

messages = [
    system_msg,
    HumanMessage("How do I create a REST API?")
]
response = model.invoke(messages)
```
</CodeGroup>

### Provider handling

Different [providers]() handle `SystemMessage` in various ways:

* **Native support** - Message included with `system` role
* **API parameter** - Content passed as separate system parameter  
* **No support** - Content merged into first human message

<Info>
    LangChain handles these provider differences automatically, ensuring your
    system messages work consistently across all supported models.
</Info>

## Human message

A `HumanMessage` represent user input and interactions. They can contain text,
images, audio, files, and any other multimodal [content](#content).

### Text content

<CodeGroup>
```python Message object
human_msg = HumanMessage("What is machine learning?")
response = model.invoke([human_msg])
```

```python String shortcut
# Using a string is a shortcut for a single HumanMessage
response = model.invoke("What is machine learning?")
```
</CodeGroup>


### Message metadata

<CodeGroup>
```python Add metadata
human_msg = HumanMessage(
    content="Hello!",
    name="alice",  # Optional: identify different users
    id="msg_123",  # Optional: unique identifier for tracing
)
```

```python Multi-user scenarios
messages = [
    HumanMessage("I like apples", name="alice"),
    HumanMessage("I prefer oranges", name="bob"),
    HumanMessage("What fruits do Alice and Bob like?", name="charlie")
]
response = model.invoke(messages)
```
</CodeGroup>

<Note>
    The `name` field behavior varies by provider - some use it for user
    identification, others ignore it. To check, refer to your model provider's
    [reference]().
</Note>

## AI message

An `AIMessage` represents model responses and contain the output of model
invocations. They can include multimodal data, tool calls, provider-specific
metadata.

### Basic AI messages

```python
response = model.invoke("Explain AI")
print(type(response))  # <class 'langchain_core.messages.AIMessage'>
```

### AI message attributes

```python AI message attributes
response = model.invoke("Tell me a joke")

print(f"Text: {response.text}")  # Text content
print(f"Content: {response.content}")  # Raw provider content
print(f"Content blocks: {response.content_blocks}")  # Standardized content blocks
print(f"Tool calls: {response.tool_calls}")  # (Empty list if no tools called)
print(f"ID: {response.id}")  # Auto-generated or provider message ID
print(f"Usage: {response.usage_metadata}")  # Token counts (when available)
print(f"Response metadata: {response.response_metadata}")  # Provider-specific data
```

### Tool calling responses

When models make tool calls, they're included in the AI message:

```python Tool calling
model_with_tools = model.bind_tools([GetWeather])
response = model_with_tools.invoke("What's the weather in Paris?")

if response.tool_calls:
    for tool_call in response.tool_calls:
        print(f"Tool: {tool_call['name']}")
        print(f"Args: {tool_call['args']}")
        print(f"ID: {tool_call['id']}")
```

### Streaming and chunks

During streaming, you receive `AIMessageChunk` objects that can be combined:

<CodeGroup>
```python Stream and collect chunks
chunks = []
for chunk in model.stream("Write a poem"):
    chunks.append(chunk)
    print(chunk.text)
```

```python Combine chunks
from langchain_core.messages import AIMessage

# Combine chunks into final message
final_message = sum(chunks, AIMessage(""))
print(f"Full response: {final_message.text}")
```
</CodeGroup>

### Basic tool messages

```python
# After a model makes a tool call
ai_message = AIMessage(
    tool_calls=[{
        "name": "get_weather",
        "args": {"location": "San Francisco"},
        "id": "call_123"
    }]
)

# Execute tool and create result message
weather_result = "Sunny, 72°F"
tool_message = ToolMessage(
    content=weather_result,
    tool_call_id="call_123"  # Must match the call ID
)

# Continue conversation
messages = [
    HumanMessage("What's the weather in San Francisco?"),
    ai_message,  # Model's tool call
    tool_message,  # Tool execution result
]
response = model.invoke(messages)  # Model processes the result
```

### Tool message structure

```python Tool message with metadata
tool_message = ToolMessage(
    content="Operation completed successfully",
    tool_call_id="call_123",  # Links to specific tool call
    name="calculator",  # Optional tool name
    artifact={"raw_data": [1, 2, 3]}  # Additional data not sent to model
)
```

<Note>
    The `artifact` field stores supplementary data that won't be sent to the
    model but can be accessed programmatically. This is useful for storing raw
    results, debugging information, or data for downstream processing without
    cluttering the model's context.

    <Accordion
        title="Example: Using artifact for raw data"
    >
        ```python
        from langchain_core.messages import ToolMessage
        import json

        # Tool execution returns structured data
        raw_data = {"temperature": 72, "condition": "Sunny"}
        tool_message = ToolMessage(
            content="The weather is sunny with a temperature of 72°F.",
            tool_call_id="call_123",
            name="get_weather",
            artifact={"raw_data": raw_data}  # Store structured data
        )

        # Later, access the raw data programmatically
        weather_info = tool_message.artifact.get("raw_data")
        print(f"Raw weather data: {json.dumps(weather_info)}")
        ```
    </Accordion>
</Note>

## Content

Different providers return model responses in various formats. To standardize
this, LangChain has a standard collection of **content blocks** to enable
**provider-agnostic** multimodal interactions. Instead of handling different
image formats, audio types, and document structures across providers,
content blocks provide a unified interface that works everywhere.

<CodeGroup>
```python Provider-specific format examples
# OpenAI format
openai_message = {
    "role": "user", 
    "content": [
        {"type": "text", "text": "What's in this image?"}, 
        {"type": "image_url", "image_url": {"url": "..."}}
    ]
}

# Anthropic format
anthropic_message = {
    "role": "user",
    "content": [
        {"type": "text", "text": "What's in this image?"},
        {"type": "image", "source": {"type": "url", "media_type": "image/jpeg", "url": "..."}}
    ]
}
```

```python Standard content blocks
universal_message = HumanMessage([
    {"type": "text", "text": "What's in this image?"},
    {"type": "image", "url": "..."}  # Works with any provider
])
```
</CodeGroup>

### Examples

```python Multimodal message
multimodal_message = HumanMessage([

    {"type": "image", "url": "chart.jpg"},
```

### Checkpointers

Checkpointers enable persistence of conversation state across sessions:

<CodeGroup>
```python In-memory checkpointer
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.graph import MessagesState, StateGraph

# Create checkpointer for development/testing
memory = InMemorySaver()

# Build your graph
builder = StateGraph(MessagesState)
# ... add nodes and edges ...
graph = builder.compile(checkpointer=memory)

# Invoke with thread ID for conversation continuity
config = {"configurable": {"thread_id": "user_123"}}
```

```python Rolling summarization
from langchain_core.messages import SystemMessage
from langchain_core.prompts import ChatPromptTemplate

class ConversationSummarizer:
    """Progressively summarize conversations"""
    
    def __init__(self, model, chunk_size=10):
        self.model = model
        self.chunk_size = chunk_size
        self.summaries = []
    
    def add_messages(self, messages):
        """Add messages and summarize if needed"""
        if len(messages) >= self.chunk_size:
            # Summarize the chunk
            summary = self.summarize_chunk(messages)
            self.summaries.append(summary)
            
            # Return summary + recent messages
            recent = messages[-3:]  # Keep last 3 messages
            return [self.get_combined_summary()] + recent
        return messages
    
    def summarize_chunk(self, messages):
        """Summarize a chunk of messages"""
        prompt = ChatPromptTemplate.from_messages([
            ("system", "Summarize these messages concisely:"),
            ("human", "{messages}")
        ])
        
        messages_text = "\n".join([f"{m.type}: {m.text}" for m in messages])
        response = self.model.invoke(
            prompt.format_messages(messages=messages_text)
        )
        return response.text
    
    def get_combined_summary(self):
        """Get combined summary of all chunks"""
        if not self.summaries:
            return None
        
        combined = " ".join(self.summaries)
        return SystemMessage(
            f"Conversation context: {combined}",
            metadata={"type": "rolling_summary"}
        )
```

```python Remove invalid messages
    for msg in messages:
        if isinstance(msg, ToolMessage):
            # Only keep tool messages with valid IDs
            if msg.tool_call_id in tool_call_ids:
                cleaned.append(msg)
        else:
            cleaned.append(msg)
    
    return cleaned
```

                ```python
                import uuid
                from datetime import datetime
                from langchain_core.messages import HumanMessage, AIMessage
                
                class VersionedMessage:
                    """Track message edit history"""
                    
                    def __init__(self, content, role="human"):
                        self.id = str(uuid.uuid4())
                        self.role = role
                        self.versions = [
                            {
                                "content": content,
                                "timestamp": datetime.now(),
                                "version": 1
                            }
                        ]
                    
                    def edit(self, new_content):
                        """Add new version of message"""
                        self.versions.append({
                            "content": new_content,
                            "timestamp": datetime.now(),
                            "version": len(self.versions) + 1
                        })
                    
                    @property
                    def content(self):
                        """Get current content"""
                        return self.versions[-1]["content"]
                    
                    def get_version(self, version_num):
                        """Get specific version"""
                        for v in self.versions:
                            if v["version"] == version_num:
                                return v["content"]
                        return None
                    
                    def to_langchain_message(self):
                        """Convert to LangChain message"""
                        msg_class = HumanMessage if self.role == "human" else AIMessage
                        return msg_class(
                            content=self.content,
                            id=self.id,
                            metadata={
                                "version": len(self.versions),
                                "edited": len(self.versions) > 1
                            }
                        )
                ```

                ```python
                class DiffTrackedMessage(VersionedMessage):

                        """Get diff between two versions"""
                        if v1 is None:
                            v1 = len(self.versions) - 1
                        if v2 is None:
                            v2 = len(self.versions)
                        
                        content1 = self.get_version(v1)
                        content2 = self.get_version(v2)
                        
                        differ = difflib.unified_diff(
                            content1.splitlines(),
                            content2.splitlines(),
                            lineterm='',
                            n=3
                        )

                                "from_version": i,
                                "to_version": i + 1,
                                "timestamp": self.versions[i]["timestamp"],
                                "diff": self.get_diff(i, i + 1)
                            })
                ```

                ```python
                class ConversationTree:
                    """Support branching conversations"""
                    
                    def __init__(self):
                        self.branches = {}
                        self.current_branch = "main"
                    
                    def add_message(self, message, branch=None):
                        """Add message to branch"""
                        branch = branch or self.current_branch
                        if branch not in self.branches:
                            self.branches[branch] = []
                        self.branches[branch].append(message)
                    
                    def create_branch(self, from_branch="main", at_index=-1):
                        """Create new branch from existing conversation"""
                        new_branch_id = f"branch_{len(self.branches)}"
                        
                        # Copy messages up to branch point
                        source_messages = self.branches[from_branch][:at_index]
                        self.branches[new_branch_id] = source_messages.copy()
                        
                        return new_branch_id
                    
                    def merge_branches(self, branch1, branch2, strategy="append"):
                        """Merge two conversation branches"""
                        if strategy == "append":
                            # Simply append branch2 to branch1
                            merged = self.branches[branch1] + self.branches[branch2]
                        elif strategy == "interleave":
                            # Interleave messages by timestamp
                            merged = sorted(
                                self.branches[branch1] + self.branches[branch2],
                                key=lambda m: m.metadata.get("timestamp", 0)
                            )
                        
                        return merged
                    
                    def get_messages(self, branch=None):
                        """Get messages from branch"""
                        branch = branch or self.current_branch
                        return self.branches.get(branch, [])
                ```

## Content output versions: v0 vs v1

LangChain v1.0 introduces standardized content formats while maintaining
backward compatibility with existing code.

### Version comparison

| Aspect | v0 (Legacy) | v1 (Standard) |
|--------|-------------|---------------|
| **Default behavior** | Yes (for compatibility) | Opt-in via `output_version` |
| **Content format** | Provider-specific | Standardized `ContentBlock` types |
| **Type safety** | Limited | Fully typed |
| **Multimodal support** | Basic | Comprehensive |

### Setting output version

<CodeGroup>
```python Model initialization
# Default v0 (backward compatible)
model = ChatOpenAI()

# Explicit v1
model = ChatOpenAI(output_version="v1")
```

```python Per invocation
# Override at invocation time
response = model.invoke(
    "Hello",
    output_version="v1"  # Override model default
)
```

```python Environment variable
import os
os.environ["LC_OUTPUT_VERSION"] = "v1"

```

### Accessing content blocks

